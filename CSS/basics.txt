
# CSS (Cascading Style Sheets)

## What is CSS?
CSS is a stylesheet language used to control the appearance and presentation of HTML elements on a webpage.
HTML creates the structure, while CSS defines the visual style, spacing, color, typography, and layout.

## Why use CSS?
- Improves appearance with colors, backgrounds, fonts, borders, and layout.
- Separates HTML content from visual design.
- Keeps styles consistent across multiple pages.
- Makes maintenance easier through reusable rules.
- Helps build responsive designs for all screen sizes.

## Types of CSS

### 1. Inline CSS
CSS is written directly inside an HTML element.

Example:
<p style="color: red;">Hello World</p>

Use cases:
- Quick testing
- Temporary styling
- Email templates

### 2. Internal CSS
CSS is written inside a `<style>` block in the document head.

Example:
<style>
  p {
    color: red;
  }
</style>

Use cases:
- Small websites
- Single-page demos
- Teaching examples

### 3. External CSS
CSS is written in a separate `.css` file and linked from HTML.

Example:
style.css
```
p {
  color: red;
}
```
HTML:
```
<link rel="stylesheet" href="style.css">
```
Use cases:
- Professional websites
- Multi-page sites
- Larger projects

## CSS Priority (Precedence)
When the same property is defined in multiple places, the browser chooses the rule with higher precedence.

Normal priority order:
1. Inline CSS
2. Internal CSS
3. External CSS

If two rules have the same specificity, the rule declared later wins.

Example:
<link rel="stylesheet" href="style.css">
<style>
  h1 {
    color: red;
  }
</style>

Result: the heading is red because the internal CSS appears after the external file.

Example when external CSS appears later:
<style>
  h1 {
    color: red;
  }
</style>
<link rel="stylesheet" href="style.css">

If `style.css` contains:
```
h1 {
  color: blue;
}
```
Then the heading becomes blue.

Rule to remember:
**When specificity is the same, the rule declared later wins.**

## What are Selectors?
Selectors tell CSS which HTML elements should receive styling.

General syntax:
```
selector {
  property: value;
}
```

Example:
```
h1 {
  color: blue;
}
```

## Types of Selectors

### 1. Element Selector
Targets all elements of a specific HTML tag.

Example:
```
p {
  color: red;
}
```
Use case: style all paragraphs in the same way.

### 2. ID Selector
Starts with `#` and targets one unique element.

Example:
```
#heading {
  color: blue;
}
```
Use case: a single page element like a title or navigation container.

### 3. Class Selector
Starts with `.` and targets one or more elements.

Example:
```
.box {
  background-color: yellow;
}
```
Use case: reusable styles for buttons, cards, or section blocks.

### 4. Universal Selector
Selects every element on the page.

Example:
```
* {
  margin: 0;
  padding: 0;
}
```
Use case: reset default browser spacing and create a clean base.

## Applying Properties
Multiple properties can be grouped inside one selector block.

Example:
```
h1 {
  color: blue;
  background-color: yellow;
  text-align: center;
}
```

## Color Property
Changes the text color of an element.

Syntax:
```
color: value;
```

Supported formats:
- Named colors: `red`, `blue`, `green`
- RGB: `rgb(255, 0, 0)`
- HEX: `#FF0000`
- HSL: `hsl(0, 100%, 50%)`

Examples:
```
h1 {
  color: #1f3a93;
}
p {
  color: rgb(50, 50, 60);
}
a {
  color: hsl(210, 80%, 40%);
}
```

## Background Property
Controls the background appearance of elements.

Examples:
```
background-color: yellow;
background-color: rgb(255, 255, 0);
background-color: #FFFF00;
background-color: hsl(60, 100%, 50%);
```

You can also use gradients:
```
background: linear-gradient(90deg, #61d0a7, #389ed9, #0a2baf);
```
